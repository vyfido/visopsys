HOW TO INSTALL VISOPSYS USING A UNIX SYSTEM

OVERVIEW

This file contains detailed instructions for installing Visopsys if you are currently running some version of UNIX. The files in this directory are the tools you will need for doing this.

INSTALLATION INSTRUCTIONS

Your top-level Visopsys directory should contain the following:

README.TXT  General README file.
dosutil Utilities for installing Visopsys using a DOS/Windows system
instfile Contains the Visopsys files that get installed
unixutil This directory

NOTE that the UNIX installation tools have been designed specifically with and for Linux. They will probably work for other UNIX systems in the future, and may only need a small amount of tweaking to work in their current form. Additionally Your UNIX system must be able to support MS-DOS filesystems; specifically the 12-bit FAT filesystem commonly used with DOS floppy disks (Linux supports this). 

To install Visopsys, cd to the directory where you unpacked Visopsys. Next, change to the "unixutil" directory. There you should see something similar to the following:

total 20
drwxr-xr-x 2 andymc 4096 Apr 1 23:13 ./
drwxr-xr-x 5 andymc 4096 Apr 1 23:13 ../
-rw-r--r-- 1 andymc 3933 Apr 1 23:13 INSTALL.TXT
-rwxr-xr-x 1 andymc 3690 Apr 1 23:13 floppy-install.csh*
-rwxr-xr-x 1 andymc 1355 Apr 1 23:13 platform.csh*
andy-linux%

First, you need to insert a floppy disk in your first floppy drive. Please use a floppy disk that is empty or conatins unwanted data, as this installation process will ERASE ALL DATA ON THE FLOPPY WITHOUT ANY FURTHER PROMPTING. Make sure the floppy disk is not write-protected (see the little movable tab on the back side of the floppy disk -- the tab should cover the hole).

The file you need to run to install Visopsys is floppy-install.csh. Before doing so, you will need to ensure a couple of things:

  1. The UNIX account you're using must have permission to mount the floppy disk in the "usual" place (see your /etc/fstab, /etc/vfstab, /etc/checklist, or whatever is appropriate for your version of UNIX).
  2. The UNIX account you're using must have permission to write directly to the floppy disk device file, such as /dev/fd0. 

If you can become root on your system, this should provide all the permissions you need. Note that the installation script assumes that the  device file for your floppy drive is /dev/fd0 (common on Linux systems).  If this is not the correct device file for your system, you will need to edit the floppy-install.csh script and make the appropriate change.  When you run floppy-install.csh, you should see output similar to the following:

FLOPPY-INSTALL - Visopsys floppy disk installation script for UNIX
Copyright (C) 2000 Andrew McLaughlin
-= NOTE: suitable for use only with Visopsys BINARY distribution =-

Making Visopsys boot diskette in /dev/fd0
Reading floppy device info from /etc/fstab
/sbin/mkdosfs 2.2 (06 Jul 1999)
Installing boot sector... Done
Copying os loader... Done
Copying kernel... Done

The entire process should take about 15 seconds. If you do not see any unusual error messages, then Visopsys is now installed on your floppy disk.  You should now be able to reboot your system using the Visopsys floppy you've just created. Tip: make sure your system BIOS is set to be bootable from a floppy. If you insert the floppy and reboot the system, but do not see messages about Visopsys, you need to enter your BIOS setup and change your "boot sequence" to something like A,C or A,CDROM,C (this is BIOS-specific).   The point is that the computer should first attempt to boot from the floppy drive where you insert the Visopsys boot diskette.

Good luck!
