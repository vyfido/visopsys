DELAYED INITIAL RELEASE - EXPLANATION

The previously completed development version of Visopsys did not support virtual memory, paging, or memory protection (see this page for more information).  Additionally, the kernel's Application Programming Interface (API) was too limited to be of much use in application development.  In order for the first public version (0.1) of Visopsys to be more compelling to potential developers, it has been determined that these capabilities should be included in the initial release.

These features have been implemented, and the resulting delay in the release schedule should be short.  Stay tuned for a new release date.
